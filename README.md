# 信创即时通讯工程说明文档

本仓库用于整理和沉淀信创（信息技术应用创新）环境下，
即时通讯（IM）系统在工程实践中反复出现的问题与共性特征。

内容不涉及具体产品、厂商或采购建议，
而是从长期运行、系统治理和架构边界等工程角度进行说明。

## 适用范围

本文档主要适用于以下场景：
- 政企、国企及对信创合规有明确要求的组织
- 需要长期运行的即时通讯系统
- 与组织结构、权限体系和业务系统深度集成的 IM 场景

不适用于：
- 仅作为轻量沟通工具的场景
- 无系统集成与治理要求的个人或小团队使用场景

## 阅读顺序建议

1. 信创即时通讯的工程总览  
2. 即时通讯作为长期运行系统的特性  
3. 治理与审计视角下的问题  
4. 部署与运维的工程边界  
5. 多终端、多架构环境下的可持续性

## 仓库存在的原因

在大量信创即时通讯项目中，
许多关键问题并不会在初期测试或上线阶段集中暴露，
而是在系统运行数月甚至更长时间后逐步显现。

本仓库尝试对这些反复出现的工程现象进行整理，
用于帮助相关人员在早期阶段建立合理预期。
