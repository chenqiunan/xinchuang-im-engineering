# 多终端与多架构环境下的可持续性问题

## 真实环境中的终端多样性

在信创场景中，
即时通讯系统通常需要同时支持：
- Windows、macOS
- UOS、银河麒麟等国产操作系统
- x86、ARM64、LoongArch 等 CPU 架构

## “能运行”并不是工程终点

真正的工程挑战不在于系统是否可运行，
而在于：
- 各终端行为是否一致
- 客户端是否易于维护
- 演进成本是否可控

## 构建与发布复杂度

终端和架构的增加，
会带来：
- 构建链路复杂化
- 发布协调成本上升
- 定位问题难度增大

## 可持续性的工程视角

多终端、多架构支持，
本质上是长期工程能力的体现，
其风险往往在初期被低估。
